1. Access requirements

The Fred Learning Review dashboard is restricted to users with the Admin role. The application has three access levels:

Role How it's assigned Access
User Default for all accounts Product selection, PDF generation, Fred AI chat
Staff Automatic for @seima.com.au emails, or set explicitly All User features + Validator
Admin Set explicitly via Google Sheet Role column All Staff features + Fred Learning Review

Granting Admin access to another user

  1. Open the Users sheet in the Google Apps Script backend spreadsheet.
  2. Find the user's row by their email address.
  3. In the Role column, enter admin (case-insensitive).
  4. The user will see the Fred Review button in the navigation bar on their next login.

2. How learning works

Fred's learning system collects feedback from users to improve response quality over time. Here is the flow:

  1. User asks a question — the question text is logged locally in the browser.
  2. Fred responds — thumbs-up / thumbs-down buttons appear below each response.
  3. User rates the response — the rating (along with the question and answer text) is saved to the browser's localStorage.
  4. Sync to cloud — the next time the chat panel opens, any unsynced feedback and question counts are sent to the Cloudflare Worker, which stores them in KV (Key-Value) storage.
  5. Dynamic suggestions — the most frequently asked questions are served back to users as quick-start suggestion buttons in the chat.

Important: Fred does not currently use feedback to change its actual AI responses. The learning data is informational — it helps admins identify gaps in Fred's knowledge so you can update the system prompt, add FAQs, or fine-tune behaviour manually.

3. Using the Review dashboard

The Fred Learning Review page (the screen you came from) gives you an at-a-glance view of:

  • Summary cards — total helpful / not-helpful ratings and overall satisfaction percentage.
  • Thumbs-Down Responses — the most recent answers users found unhelpful. Use these to identify where Fred is falling short.
  • Top Questions — the most frequently asked questions across all users. These drive the dynamic suggestion buttons in the chat.
  • Recent Feedback — a full feed of all feedback entries, filterable by Up / Down.

Click the Refresh button to reload the latest data from Cloudflare.

4. Where data is stored

Browser (localStorage) — per user, per device

Each user's browser holds their own chat history and pending feedback. This data syncs to the cloud automatically and is pruned after 7 days. The keys are:

  • fredChatHistory — conversation messages (for session restore).
  • fredChatMessages — rendered message HTML (for UI restore).
  • fredFeedback — individual rating entries, each marked synced or unsynced.
  • fredQuestionLog — running tally of questions this user has asked.

Cloudflare KV — centralised, cross-user

The aggregated learning data lives in a Cloudflare Workers KV namespace called FRED_LEARNING. This is the "source of truth" that the Review dashboard reads from.

5. Reviewing data in Cloudflare

You can browse and edit the raw KV data through the Cloudflare Dashboard:

  1. Click the Cloudflare KV button in the Fred Review navbar (or go to dash.cloudflare.com → Workers & Pages → KV).
  2. You will see a list of keys. Each key holds a JSON value.
  3. Click any key to view its content. Click Edit to modify the value directly.

Tip: You must be logged in to Cloudflare with the account jim@seima.com.au to access the dashboard.

6. Editing and deleting data

Curate the top questions list

The stats:topQuestions key holds a JSON array of the top 20 most-asked questions. These are shown to users as suggestion buttons when they open the Fred chat. You can curate this list in several ways:

Remove a single question:

  1. Open the KV namespace in Cloudflare and find stats:topQuestions.
  2. The value is a JSON array like: [{"prompt": "...", "count": 5}, ...]
  3. Delete the object for the question you want to remove, then save.

Reword a question:

  1. Find the entry in the array and edit the "prompt" value to your preferred wording.
  2. This changes what users see on the suggestion button without affecting the count.

Promote or demote a question:

  1. Change the "count" value. Higher counts appear first. Set a high count to promote a question to the top.

Reset all suggestions:

  1. Edit the value to [] (empty array). Suggestions will rebuild organically as users ask questions.

Reset feedback statistics

  1. Find the key stats:feedbackSummary.
  2. Edit the value to reset counters:
{"totalUp": 0, "totalDown": 0, "recentDown": []}

Delete an individual feedback entry

  1. Individual feedback entries are stored with keys like feedback:abc123...
  2. Find the entry in the key list and click Delete.

Note: Deleting individual feedback:* entries removes them from the "Recent Feedback" list but does not automatically update the summary counters in stats:feedbackSummary. If you delete many entries, consider resetting the summary as well.

Automatic cleanup

You do not need to manually delete old feedback entries. Each feedback:{id} entry is stored with a 90-day TTL (time-to-live) and is automatically removed by Cloudflare after it expires. On the client side, synced feedback entries in localStorage are pruned after 7 days. The stats:topQuestions and stats:feedbackSummary keys do not expire and must be curated manually if needed.

7. Improving Fred's responses

When you see patterns in the thumbs-down feedback — for example, Fred consistently gives wrong answers about a specific product category or misunderstands a type of question — you can improve his behaviour by updating the system prompt.

Where the system prompt lives

Fred's instructions are defined in the file:

packages/presenter/js/ai-service.js

Look for the constant called CATALOG_PROMPT. This is the text that tells Fred who he is, what rules to follow, and how to respond. It is sent with every chat request.

What you can change

  • Add specific instructions — e.g., "When asked about freestanding baths, always mention the weight capacity."
  • Correct common mistakes — e.g., "Never recommend discontinued products from the Heritage range."
  • Add FAQ-style answers — e.g., "If the user asks about warranty, explain that all Seima products carry a 25-year warranty on..."
  • Adjust tone or formatting — e.g., "Always respond in dot points when listing multiple products."

Typical workflow

  1. Review the thumbs-down feedback on the dashboard to identify recurring issues.
  2. Open packages/presenter/js/ai-service.js in your code editor.
  3. Edit the CATALOG_PROMPT string to address the issue.
  4. Test locally by running the dev server (npm run dev:presenter) and asking Fred the same questions.
  5. Commit, push, and deploy when satisfied.

Tip: Keep the system prompt focused and specific. Broad or vague instructions tend to be ignored by the AI model. Concrete examples work best — e.g., "When comparing products, always include dimensions and price" is better than "Be helpful."

8. Using the command line (advanced)

If you prefer the terminal, you can use the Wrangler CLI to interact with KV. Run these from the packages/ai-proxy directory.

List all keys

npx wrangler kv key list --namespace-id="4118f16373a244178acd1da59a04e208"

Read a key

npx wrangler kv key get --namespace-id="4118f16373a244178acd1da59a04e208" "stats:topQuestions"

Update a key

npx wrangler kv key put --namespace-id="4118f16373a244178acd1da59a04e208" "stats:topQuestions" "[]"

Delete a key

npx wrangler kv key delete --namespace-id="4118f16373a244178acd1da59a04e208" "feedback:some-id-here"

9. KV key reference

Key Type Description
stats:topQuestions JSON array Top 20 most asked questions. Each entry: {"prompt": "...", "count": N}. Drives the chat suggestion buttons.
stats:feedbackSummary JSON object Aggregate counts: totalUp, totalDown, recentDown (last 20 thumbs-down with question/answer).
feedback:{id} JSON object Individual feedback entry. Fields: question, answer, rating (up/down), timestamp. Auto-expires after 90 days.
